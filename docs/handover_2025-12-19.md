### 2) Threshold tuning (coverage vs “uncertain” rate)

- Ran the pipeline at multiple classifier thresholds and saved separate CSVs
- Swept thresholds and produced summary JSON and confusion comparisons

### 3) Fixed two major pipeline issues that were distorting results

- **Detector semantics fix**: detector model outputs **P(not_galago)** (sigmoid for label 1), so the pipeline now uses:
  - `galago_prob = 1.0 - not_galago_prob`
- **Mean pooling dilution fix**: replaced naive mean pooling across all windows with:
  - RMS-gated window selection + **top-K pooling** (default `POOL_TOPK_WINDOWS=3`)

### 4) Fixed the major 19-class training bug (double-normalization)

- In the 19-class training scripts, images are normalized in the loader (`/255.0`), so we removed the model’s extra `Rescaling(1/255)` layer.
- Added **versioned class-name saving**: `class_names_{num_classes}.json`.
- Retrained a short run (“fixnorm”) to confirm learning resumed.

## Known caveats / gotchas (so you don’t re-hit them)

- **Don’t overwrite prediction CSVs**: always pass `--out-csv` when iterating thresholds/models.
- **Run analysis after predictions finish writing**: otherwise you can accidentally analyze a partial file.
- **Detector output is P(not_galago)**: if you ever refactor `predict_detector()`, keep this semantic straight.
- **Avoid double normalization**: either normalize in the dataset loader OR in the model, not both.
